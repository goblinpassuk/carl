🔐 How Deterministic PRF Works

  1. Register YubiKey ONCE - creates a permanent credential
  2. Credential has a unique private key that never leaves the YubiKey
  3. PRF = HMAC(private_key, domain_challenge)
  4. Same key + same challenge = same output ALWAYS
  5. HKDF converts PRF output into your password

✅ Why This Is Deterministic

  • One credential, one private key
  • HMAC is mathematically deterministic
  • Same YubiKey + same domain = same password
  • Re-registering creates a NEW key = DIFFERENT passwords

⚠️ Critical Rule

NEVER re-register unless you want all new passwords. If you lose the credential ID, you lose all your passwords. Back up your credential ID!
