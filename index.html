🔐 How It Works

This uses WebAuthn PRF (Pseudo-Random Function):

  1. Your YubiKey holds a secret that never leaves the device
  2. A challenge is derived from domain + username
  3. YubiKey computes HMAC internally, returns result
  4. Result becomes your master key
  5. HKDF derives your password from master key

🔒 Security Properties

  • Hardware-bound: Key never leaves YubiKey
  • Deterministic: Same inputs = same password
  • No storage: Passwords exist only when generated
  • Domain isolation: Unique password per site
  • Offline capable: No network calls after load

💡 Tips

  • Register once, use forever
  • Get a backup YubiKey, register it too
  • Be consistent with domain format
  • Save configurations for quick access
